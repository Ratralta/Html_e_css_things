# O que é : 
* O HTTP define como um **cliente** requisita uma [pagina web](html_diferenca_entre_pagina_web_site_servidor_web_e_mecanismo_de_busca) de um **servidor** E como o [servidor](html_servidor_web) responde a requisição. Pode ser que requisição a pagina web seja somente para um objeto especifico da pagina. 
* O [navegador web](html_diferenca_entre_pagina_web_site_servidor_web_e_mecanismo_de_busca) pode ser um **cliente**, o [servidor web](html_servidor_web) é o **servidor**.
* É usado em [API's rest](html_API). 

# Request e Response :  
## [-- Link --](https://www.youtube.com/watch?v=NZoFcr7R4zQ)
## Request :
* O request é uma "mensagem de requisição HTTP" para um **objeto** duma **"pagina web"**
* O **cliente** manda a mensagem para o **servidor**. 
* **GET** geralmente irá **buscar** uma informação. 
* **POST** geralmente irá **adicionar** uma informação. 
* EX :
	![[HTTP_request.png]]
## Response : 
* É o retorno do servidor de um **request**.
* Sendo o retorno uma "mensagem HTTP" que contem os **objetos solicitados**.
* EX
	![[HTTP_response.png]]
	* **Status Line** : Mostra se a requisição deu certo.

# Métodos principais : 
* **GET** : Pede uma informação para ser recebida. 
* **POST** : Adiciona uma informação. 
* **PUT** : Atualiza alguma informação que já existe. 
* **DELETE** : Deleta alguma informação.
